# Written Proposal

## Problem 
Searching for a recipe online is hard, and oftentimes you don’t know what is something new that you can cook with the existing ingredients and time constraints given. A lot of times, the results provided online are limited towards a few key words and are largely not sensitive towards particular semantics. Therefore, we are going to create a recipe finder that is semantic-aware, able to accommodate a wide range of diets and can meaningfully improve the culinary experience. We will be basing off a dataset from Food.com which contains 230,000 recipes. Each recipe contains a name, ingredient list, cooking steps, tags, a nutrition vector (calories, fat, sugar, sodium, protein, saturated fat, and carbohydrates) and a description for each dish. Alongside, is a 700,000 row database that contains reviews and ratings towards the recipes. The description and ratings of each dish will allow us to create a base towards generating a semantically aware search application that can take on the different queries by the user. The time, ingredients and nutrition vector will allow us to make certain accommodations and vectors towards different diets and constraints provided. In order to make the web-app as customizable as possible, we plan to add several features to create the most customized dish. For example, we customize dietary restrictions, nutritional emphasis, and spiciness. If we were to also extend this, we could also look into APIs on food price to create an expected cost per dish or allow images as input or as an output to help visualize the user experience. .We believe that this database will help us create an interactive web application to help the user search for the best recipe. 

## Methods
For this project, we will use a wide variety of algorithms to create the web-app. We will first clean the recipe and encode each of them into dense embedding in order to transform such complex high-dimensional datasets into lower dimensional numerical vectors. This will enable the model to learn each recipe more efficiently and effectively. To identify the most relevant recipe according to the user’s query, we will employ cosine similarity to semantically retrieve the recipe. Cosine similarity measures the directional alignment between vectors rather than their magnitude, making it well-suited for matching a user's query to recipes by meaning rather than exact keyword overlap. We will also implement k-means clustering from scratch in PyTorch to create clusters of similar recipes that users can take inspiration from. We will also fit a regression model to create a rating-based ranking signal in order to help the user find the most optimal recipe based on a multitude of factors including ratings, time, nutritional values and more. We can try to do a semantic analysis of the reviews, and for ratings, since most ratings are skewed towards 5, we will need to consider a form of regularization before it can be added as a feature to the dataset. To address rating reliability, we aggregate user ratings from the interactions dataset using a Bayesian average that shrinks recipes with few reviews toward the global mean, ensuring that the regression model's training signal is not biased by low-review-count outliers. We will also project embeddings into a 2D with PCA and a non-linear autoencoder to reduce the high-dimensional recipe embeddings to two dimensions for an interactive scatter-plot visualization in the app, allowing users to see how recipes are distributed in semantic space and how clusters relate to one another. We will lastly, implement all of these algorithms and visualise them into a streamlit app.
